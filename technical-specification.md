Техническое задание на разработку системы автоматизации учета заказов и рассылки счетов в WhatsApp

1. Общее описание
Компания занимается продажей растений через группы в мессенджере WhatsApp. Текущий процесс учета заказов, формирования и отправки счетов клиентам полностью ручной, что приводит к большим временным затратам и риску ошибок.
Цель проекта — разработать программное решение, которое автоматизирует процесс сбора данных о заказах из таблицы Google Sheets, рассчитывает итоговую сумму для каждого клиента и автоматически отправляет персонализированные счета в личные чаты клиентов в WhatsApp.

2. Описание текущего бизнес-процесса (AS-IS)
Текущий процесс состоит из трех основных этапов:
Проведение распродажи в группе WhatsApp:
В группу отправляются фотографии товаров (растений) с названиями и ценами.
Клиенты бронируют товар, отправляя ответное сообщение «Бронь» на конкретную фотографию.
Ручной учет заказов в Google Sheets (Excel):
Сотрудник вручную отслеживает сообщения с бронями в WhatsApp.
Для каждого клиента создается или находится строка в общей таблице. Клиент идентифицируется по имени (если он сохранен в контактах) и последним четырем цифрам номера телефона.
В строку клиента последовательно вносятся все забронированные им товары и их цены.
Ручное формирование и отправка счетов (самый трудоемкий этап):
После окончания распродажи сотрудник поочередно обрабатывает каждого клиента из таблицы.
Находит клиента в WhatsApp (по имени или ком по номеру телефона).
Копирует заранее подготовленный текстовый шаблон для счета.
Вставляет шаблон в чат с клиентом.
Возвращается в таблицу, копирует строку с заказом данного клиента.
Вставляет скопированные данные о заказе в сообщение в WhatsApp.
Для расчета итоговой суммы, копирует цены в отдельный файл или столбец и использует функцию автосуммирования.
Вручную вписывает итоговую сумму в счет.
Отправляет сформированное сообщение клиенту.
Этот процесс повторяется для 200-300 клиентов и занимает практически целый рабочий день.

3. Проблематика
Огромные временные затраты: Ручная обработка сотен клиентов занимает несоизмеримо много времени.
Высокий риск человеческой ошибки: Возможны ошибки при копировании данных, ручном подсчете суммы, отправке счета не тому клиенту.
Низкая эффективность: Процесс монотонный и требует постоянной концентрации, что приводит к быстрой утомляемости сотрудника.

4. Цели и задачи проекта
Основная цель: Сократить время на формирование и рассылку счетов с одного рабочего дня до нескольких минут.
Задачи:
Автоматизировать сбор данных о заказах по каждому клиенту из таблицы Google Sheets.
Автоматизировать расчет итоговой суммы заказа для каждого клиента.
Автоматизировать формирование текста счета на основе шаблона и данных клиента.
Автоматизировать отправку сформированного счета в личный чат клиента в WhatsApp.

5. Функциональные требования к системе
5.1. Работа с исходными данными (Google Sheets):
Система должна уметь подключаться к указанному документу Google Sheets.
Система должна считывать данные из таблицы, где каждая строка соответствует одному клиенту.
Структура таблицы должна быть стандартизирована (например: Столбец А - Имя/ID клиента, Столбец B - Полный номер телефона, Столбцы C, D, E, F... - пары "Товар 1", "Цена 1", "Товар 2", "Цена 2" и т.д.).
5.2. Обработка заказов:
Для каждой строки (клиента) система должна автоматически собирать все заказанные им товары с ценами.
Система должна корректно суммировать все цены для получения итоговой суммы заказа.
5.3. Формирование счета:
Система должна использовать настраиваемый текстовый шаблон для счета.
В шаблоне должны быть предусмотрены переменные (плейсхолдеры) для автоматической подстановки данных, например: {имя_клиента}, {список_заказов}, {итоговая_сумма}.
Список заказов должен быть отформатирован в удобном для чтения виде (например, каждый товар с новой строки).
5.4. Интеграция и отправка в WhatsApp:
Система должна быть интегрирована с аккаунтом WhatsApp, который используется для работы.
Система должна уметь находить клиента по его полному номеру телефона и отправлять ему сообщение.
Отправка должна работать как для клиентов, сохраненных в контактах, так и для тех, кто не сохранен (идентифицируется только по номеру).
5.5. Управление и запуск:
Для запуска процесса рассылки должен быть предусмотрен простой интерфейс (например, кнопка «Начать рассылку»).
Система должна иметь возможность формировать отчет о проделанной работе (например, сколько счетов успешно отправлено, по каким номерам отправка не удалась).

6. Требования к сохранению текущей инфраструктуры
WhatsApp: Решение должно работать с существующим номером и аккаунтом WhatsApp, так как на нем сосредоточена вся клиентская база и рабочие группы.
Google Sheets (Excel): Процесс сбора заказов в таблицу должен остаться без изменений, так как он является привычным и удобным.
Рабочие группы: Система не должна мешать функционированию существующих групп в WhatsApp, где проходят распродажи.

7. Ожидаемый результат
Полноценно работающее программное решение, которое после окончания распродажи и заполнения таблицы заказов позволяет в автоматическом режиме (по нажатию одной кнопки) сформировать и отправить всем клиентам корректные и персонализированные счета в WhatsApp. Процесс, занимавший ранее целый день, должен выполняться за несколько минут.

В счёте на оплату должны быть прописаны названия растений с ценами в точности с информацией из Excel.
